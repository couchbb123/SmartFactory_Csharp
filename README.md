   핵심 설계 포인트
1. Interface 기반 표준화 (iEquipment)

규격화: 모든 장비가 동일한 조작(Start, Stop)을 따르도록 인터페이스 정의

확장성: 신규 장비 추가 시 기존 코드 수정 없이 즉시 통합 가능



2. 고속 자원 관리 (Dictionary)

빠른 조회: 리스트 순회 대신 해시 테이블 기반 Dictionary 사용 → O(1) 탐색

데이터 안정성: 장비 이름을 키로 중복 등록 방지 및 데이터 무결성 확보



3. Event 기반 실시간 중계

관심사 분리: 장비는 상태를 알리기만 하고, 컨트롤러가 중앙에서 수집하여 외부 UI에 전달

자원 안정성: 재등록 시 기존 이벤트 핸들러 해제(-=)로 메모리 누수와 로그 중복 방지




4. 프로젝트 구조

iEquipment.cs: 장비 공통 규격 (Interface)

Robot.cs: 장비 동작 구현

FactoryControll.cs: 장비 등록/제어 및 이벤트 중계

Form1.cs: UI 및 실시간 모니터링
